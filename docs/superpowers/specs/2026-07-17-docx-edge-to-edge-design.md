# DOCX Edge-to-Edge Preview Design

## Goal

Remove the gray frame and 30 px outer padding generated by `docx-preview` so the white document page uses the full DOCX content width. Preserve the paper presentation, including the page shadow, document-defined page margins, and spacing between pages.

## Scope

- Apply the visual override only to rendered DOCX content.
- Keep `.doc` unsupported.
- Do not change the shared Office preview padding or the Excel and PowerPoint layouts.
- Do not change document pagination, page dimensions, or text reflow.

## Implementation

Extend the renderer-owned `DOCX_FIT_STYLE` in `office-renderers.ts` with a `.docx-wrapper` rule that sets its background to transparent and its padding to zero. The existing page rule continues to apply responsive CSS zoom and retains the vendor-provided white background, shadow, and bottom margin.

The override is injected after `docx-preview` finishes rendering, so it wins over the vendor's equally specific default wrapper rule without changing or forking the dependency. `applyDocxFit` already reads the wrapper's computed horizontal padding; after the override it will calculate the scale against the entire DOCX render target width. The existing cleanup removes the injected style, observer, pending animation frame, scale property, and generated DOM.

## Verification

- Add a renderer test that locks the transparent wrapper background and zero padding override.
- Keep the existing responsive-fit tests for widest-page selection, scale limits, resize updates, overflow alignment, and cleanup.
- Run the related Office preview tests, formatting, lint, type checks, and production build.
- Validate both the small and large DOCX fixtures from the Downloads dataset in the Electron preview panel at narrow and wide widths. Confirm that no gray frame remains, pages expand to the available content width, page shadows and inter-page spacing remain visible, and no horizontal overflow is introduced.
